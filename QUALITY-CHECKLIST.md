# Quality Checklist for Skills

Этот чек-лист обязателен для каждого skill в проекте.
Skill считается готовым только если все пункты выполнены.

---

## 1. Структура и ясность

- [ ] Есть чёткое название skill
- [ ] Есть раздел **Description** (что делает skill)
- [ ] Есть раздел **When to use** (когда применять)
- [ ] Есть раздел **Rules** / **Safety Rules**
- [ ] Есть пошаговый **Workflow** (если применимо)
- [ ] Текст написан просто и понятно (в том числе для новичка)

---

## 2. Безопасность

- [ ] Опасные действия требуют явного подтверждения пользователя
- [ ] Перед изменениями на OpenWrt всегда предлагается бэкап
- [ ] Нет команд, которые могут сломать систему без предупреждения
- [ ] Skill не противоречит system prompt

---

## 3. Источники и документация

- [ ] Указан оригинальный источник (если skill основан на архиве / другом проекте)
- [ ] Указано, что изменено по сравнению с оригиналом (если есть изменения)
- [ ] Есть приоритет локальной документации (`docs/`)
- [ ] Есть ссылки на официальную документацию (особенно для OpenWrt)

---

## 4. Техническая корректность

- [ ] Все команды реальные и актуальные
- [ ] Команды проверены по официальной документации
- [ ] Нет выдуманных или устаревших команд
- [ ] Учитывается работа с флешки (USB)

---

## 5. Совместимость с проектом

- [ ] Подходит под goclaw
- [ ] Учитывает hybrid-режим (cloud + Fabric)
- [ ] Совместим со структурой флешки
- [ ] Не создаёт ненужных зависимостей

---

## 6. Финальная проверка

- [ ] Skill прочитан «глазами новичка»
- [ ] Нет противоречий с другими skills
- [ ] Нет лишней «воды»
- [ ] Можно дать 2–3 тестовых запроса и skill на них отвечает

---

## Правило

Если хотя бы один важный пункт не выполнен — skill считается **не готовым**.
