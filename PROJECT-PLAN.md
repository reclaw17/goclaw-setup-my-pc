# Project Plan — goclaw-setup-my-pc

**Главный принцип: качество важнее скорости.**  
Мы не переходим к следующему этапу, пока текущий не прошёл проверку по чек-листу качества.

---

## 1. Цели проекта

Создать портативного AI-агента на базе официального goclaw, который:

- Запускается с USB-флешки
- Настраивает CachyOS / Arch Linux и Windows 11
- Настраивает OpenWrt-роутеры
- Работает в hybrid-режиме (облако → локальный Fabric при отсутствии интернета)
- Имеет локальную offline-документацию
- Соблюдает высокий уровень безопасности (Cursor-style)
- Поддерживает русский и английский языки

---

## 2. Принципы качества

1. Каждый skill проходит **QUALITY-CHECKLIST.md**
2. Опасные действия всегда требуют явного подтверждения
3. Приоритет локальной документации над интернетом
4. Официальная документация — источник правды (особенно OpenWrt)
5. Оригиналы skills из архива используются как основа, но адаптируются под наши требования
6. Лучше 5 сильных skills, чем 15 слабых
7. Не перескакиваем этапы

## 4. План по этапам

### Этап 1. Качество ключевых skills (текущий приоритет)

**Цель:** довести 4–5 самых важных skills до высокого уровня.

Порядок работы:

1. **local-docs**  
   Основа: `chip-docs-local`  
   Задача: сделать максимально близким к оригиналу + добавить приоритет USB и offline

2. **safety-confirm**  
   Усилить до уровня Cursor-style (явные подтверждения, прозрачность)

3. **openwrt**  
   Уже хороший. Дошлифовать по чек-листу и убедиться, что всё опирается на официальные доки

4. **fabric-offline**  
   Создать с нуля тщательно (запуск Fabric с флешки, работа на 8 ГБ VRAM)

5. **pc-setup**  
   Создать (CachyOS/Arch + Windows 11)

**Критерий готовности этапа:**  
Все перечисленные skills проходят QUALITY-CHECKLIST.md без замечаний.

---

### Этап 2. Offline-режим

- Интеграция Fabric как sidecar
- Выбор и подготовка модели (Qwen 3.5 9B или аналог под 8 ГБ VRAM)
- Тестирование запуска без интернета
- Корректное сообщение пользователю о режиме (online / offline)

**Критерий готовности:**  
Агент успешно переключается на локальную модель при отсутствии интернета.

---

### Этап 3. Launcher

- `launcher/start-linux.sh`
- `launcher/start-windows.bat`
- Автоопределение ОС
- Автопроверка интернета
- Автозапуск Fabric при необходимости

**Критерий готовности:**  
С флешки можно запустить агента одной командой/кликом на Linux и Windows.

---

### Этап 4. Локальная документация

- Наполнение `docs/` (OpenWrt, CachyOS, Windows 11, AdGuard, Amnezia)
- Использование подхода chip-docs-local
- Возможность пользователю добавлять свои документы в `docs/custom/`

**Критерий готовности:**  
Агент может отвечать на основные вопросы по целевым системам без интернета.

---

### Этап 5. Первый MVP

- Реальный запуск с флешки
- Проверка сценариев:
  - Настройка ПК (базовая)
  - Работа с OpenWrt (бэкап + простые изменения)
  - Offline-режим
  - Подтверждения безопасности

**Критерий готовности:**  
Агент реально помогает пользователю без критических ошибок.

---

### Этап 6. Публичный релиз

- Удаление всех секретов
- Финальный README
- Лицензия (MIT)
- Очистка временных файлов

---

## 5. Правила работы

1. Всегда работаем **по порядку**.
2. Перед переходом на следующий этап — проверка по QUALITY-CHECKLIST.md.
3. Любой новый skill сначала проходит чек-лист.
4. Предпочитаем адаптацию сильных оригиналов созданию с нуля.
5. Качество важнее скорости.

---

## 6. Открытые риски

- Размер локальной модели на флешке (нужно место)
- Совместимость Fabric с разными GPU
- Отсутствие опыта разработки у владельца проекта (поэтому все решения максимально простые и прозрачные)
- Необходимость реального тестирования на железе 8 ГБ VRAM
