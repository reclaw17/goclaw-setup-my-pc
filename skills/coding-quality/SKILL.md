# coding-quality

**Status:** Production-ready  
**Last update:** 2026-07-22

## Purpose

Ensure that any code or scripts generated by the agent are clean, correct and safe.

## When to use

- When generating shell scripts, configuration files or code
- When reviewing existing scripts
- When the user asks for automation

## Rules

1. Prefer simple, readable solutions.
2. Never generate destructive commands without confirmation.
3. Always show the full code before applying it.
4. Prefer official tools and idiomatic style for the target system.
5. For Arch/CachyOS prefer pacman/yay style, for Windows prefer winget/PowerShell.

## Integration

- Used by `pc-setup` and `openwrt` when scripts are needed
- Must go through `safety-confirm` before execution
