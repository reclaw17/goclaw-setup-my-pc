# Audit 2026-07-24

Sources reviewed:

1. Archive `human20mcp-20260722.zip` (`h20-curs` + 235 skill cards)
2. Upstream GoClaw (`nextlevelbuilder/goclaw` README, docs/07, docs/14–15, install-lite.sh)
3. This repo + `goclaw-lite-cachyos`

## Findings

### What we got right

- **Linux portable Lite via AppImage** fills a real upstream gap: official `install-lite.sh` is **macOS only** and prints *"For Linux: not yet supported"*.
- USB layout + fetch scripts (no 120MB binary in git) matches a portable product.
- Safety-first skills (confirm, admin-check) match Human20 / Cursor-style workshop culture in `h20-curs`.
- Cloud-primary + local Fabric fallback matches the product brief.

### Gaps / mistakes

| # | Issue | Severity |
|---|--------|----------|
| 1 | Our `skills/*/SKILL.md` are **agent procedure notes**, not **GoClaw-native skills** (YAML frontmatter + loader dirs per docs/14–15) | High |
| 2 | Human20 cards are **catalog entries** (slug + GitHub URL + zip). We paraphrased them instead of documenting *install from upstream zip/GitHub* | High |
| 3 | Missing GoClaw **bootstrap** files (`AGENTS.md`, `SOUL.md`, `TOOLS.md`, …) that upstream injects into the agent prompt | High |
| 4 | Docs over-promised **Windows GUI GoClaw**; official Lite installer is macOS; Windows = zip from releases when published — currently no `lite-v*` assets in recent releases | Medium |
| 5 | `docs/**` offline notes are thin vs `chip-docs-local` (real mirrors + FILELIST). OK for MVP, not yet “chip-docs-local quality” | Medium |
| 6 | Skills not auto-mounted into GoClaw workspace / managed skills dir | Medium |
| 7 | Duplicate / legacy Russian entrypoints + old `goclaw/goclaw-linux` paths add confusion | Low |

### Official GoClaw facts (do not contradict)

- **Standard** = PostgreSQL multi-tenant server (not USB-friendly).
- **Lite desktop** = SQLite, no Docker/Postgres; install script **darwin only**; Linux not supported upstream → **our AppImage is the Linux vehicle**.
- Skills: filesystem layout with `SKILL.md` frontmatter, seed into DB, `skill_search` / `use_skill`.
- Bootstrap: `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `BOOTSTRAP.md`.
- Public docs: https://docs.goclaw.sh

### Human20 / h20-curs facts

- 235 skills as **cards**, not full vendored skill trees.
- Priority slugs for this product still valid: `chip-docs-local`, `auto-model-router`, `backup-manager`, `clawguard`, `workshop-shaw-coding-standard`, `repo-task-proof-loop`, `workshop-perplex-search`, …
- Full skill bodies live on **GitHub / Human20 zips** — project should *link + optional fetch*, not pretend stubs are originals.

## Decision

1. Keep AppImage factory + USB shell.  
2. Add bootstrap markdown for Lite.  
3. Mark skills as “procedure pack v1”; add frontmatter + install path toward GoClaw loader.  
4. Document honest Windows/macOS/Linux matrix from upstream.  
5. Later: script to fetch selected Human20 skill zips into `skills/vendor/`.
